重现了上述场景出现的错误。不是会自动装箱吗？

下面从两个方面讨论。首先是Java方法调用时的方法签名，另一个是Java中的自动拆装箱机制。

方法签名

第一个问题。先跳过类这个层面，在java中之所以能调用到一个类中不同的方法，正是因为方法的签名。方法的名字和方法的参数唯一的确定了一个方法。在字节码的层面，方法的返回类型也作为了方面签名的一部分，所以在class文件中，两个相同方法名，相同参数，不同返回类型的方法是可以进行重载的。

int和Integer虽然有着千丝万缕的联系，但归根结底这两个是两个不同的类型。所以在作为方法签名的时候，两个类型的方法签名是不同的。上述例子中，B类修改重新编译后，A类中还是指向了int参数的test()方法。而此时这个方法已经不存在了，所以报出了上述的异常。
